# E2E テストシナリオ: 004 ふたりの掲示板

## 概要

| 項目 | 内容 |
|------|------|
| シナリオID | 004 |
| 機能 | ふたりの掲示板（投稿・リアクション） |
| 対象ページ | `/trips/:tripId/board` |
| 関連API | `GET /api/trips/{tripId}/board`, `POST /api/trips/{tripId}/board`, `POST /api/trips/{tripId}/board/{postId}/reactions` |
| 前提条件 | アプリ起動済み、ログイン済み、旅行が1件以上存在（tripId=1） |
| テストユーザー | たろう (`taro@example.com` / `password`)、はなこ (`hanako@example.com` / `password`) |

---

## 前提条件

- フロントエンド（Nuxt）が `http://localhost:3900` で起動していること
- バックエンド（Laravel）が `http://localhost:8900` で起動していること
- `php artisan migrate --seed` が実行済みで、テストユーザーが存在すること
- ログイン済みの状態で開始すること（TC-001-02 の手順でたろうとしてログイン済み）
- 旅行が1件以上存在し、tripId=1 でアクセス可能であること

| # | ケース名 | 種別 | 概要 |
|---|---------|------|------|

## TC-004-01: 掲示板ページの表示

### 目的

掲示板ページが正しくレンダリングされ、タイトル・投稿入力欄・送信ボタンが全て表示されることを確認する。

### 手順

```
Step 1: 掲示板ページに遷移する
  Tool: browser_navigate
  Params: { "url": "http://localhost:3900/trips/1/board" }
  期待結果: ページが正常にロードされる

Step 2: ページのスナップショットを取得して UI 要素を確認する
  Tool: browser_snapshot
  期待結果: 以下の要素が存在する
    - タイトル「ふたりの掲示板」（h1 要素）
    - テキストエリア（placeholder="メッセージを入力..."）
    - 送信ボタン（丸型の FAB ボタン、type="submit"）
    - 投稿がない場合は「まだ投稿がありません」のエンプティステートが表示される
```

### 判定基準

- ページタイトルに「ふたりの掲示板」が表示される
- 投稿入力欄（textarea）が画面下部に固定表示されている
- 送信ボタンが表示されている
- default レイアウト（ヘッダー・ナビあり）で表示されている

## TC-004-02: メッセージの投稿

### 目的

テキストメッセージを投稿し、投稿一覧にユーザー名・本文・タイムスタンプ付きで表示されることを確認する。

### 手順

```
Step 1: 掲示板ページに遷移する
  Tool: browser_navigate
  Params: { "url": "http://localhost:3900/trips/1/board" }
  期待結果: 掲示板ページが表示される

Step 2: スナップショットを取得してフォーム要素を確認する
  Tool: browser_snapshot
  期待結果: テキストエリアと送信ボタンが確認できる

Step 3: テキストエリアにメッセージを入力する
  Tool: browser_click
  Params: { "element": "テキストエリア（placeholder='メッセージを入力...'）" }
  ※ スナップショットの ref 番号を使用する

  Tool: browser_type
  Params: { "text": "明日の集合場所はどこにする？" }
  期待結果: テキストエリアに値が入力される

Step 4: 送信ボタンをクリックする
  Tool: browser_click
  Params: { "element": "送信ボタン（丸型 FAB）" }
  ※ スナップショットの ref 番号を使用する
  期待結果: 投稿処理が開始される

Step 5: 投稿結果を確認する
  Tool: browser_snapshot
  期待結果:
    - 投稿一覧に新しい投稿が表示される
    - 投稿カードにユーザー名（たろう）が表示される
    - 投稿カードに本文「明日の集合場所はどこにする？」が表示される
    - 投稿カードにタイムスタンプ（M/D HH:mm 形式）が表示される
    - 自分の投稿なので ml-8（右寄せ）で表示される
    - テキストエリアがクリア（空）になっている
    - 「まだ投稿がありません」のエンプティステートが消えている
```

### 判定基準

- 投稿が一覧に表示される（ユーザー名、本文、タイムスタンプの3要素が揃っている）
- 投稿後にテキストエリアがクリアされる
- 自分の投稿は右寄せ（ml-8）で表示される
- エラーメッセージが表示されない

## TC-004-03: 空メッセージの送信防止

### 目的

テキストエリアが空の状態では送信ボタンが無効（disabled）になり、空メッセージの送信が防止されることを確認する。

### 手順

```
Step 1: 掲示板ページに遷移する
  Tool: browser_navigate
  Params: { "url": "http://localhost:3900/trips/1/board" }
  期待結果: 掲示板ページが表示される

Step 2: スナップショットを取得して送信ボタンの状態を確認する
  Tool: browser_snapshot
  期待結果:
    - テキストエリアが空の状態である
    - 送信ボタンが disabled 状態である（disabled:bg-gray-300 クラスが適用されている）
    - 送信ボタンがクリックできない状態である

Step 3: 空白のみ入力して送信ボタンの状態を確認する
  Tool: browser_click
  Params: { "element": "テキストエリア（placeholder='メッセージを入力...'）" }
  Tool: browser_type
  Params: { "text": "   " }

Step 4: スナップショットを取得する
  Tool: browser_snapshot
  期待結果:
    - 空白のみの入力では送信ボタンが依然として disabled 状態である
    - ※ newBody.trim() が空であるため disabled が維持される
```

### 判定基準

- テキストエリアが空のとき送信ボタンが disabled になる
- 空白のみの入力でも送信ボタンが disabled のまま
- API リクエストが発生しない

## TC-004-04: リアクションの追加

### 目的

投稿の + ボタンから絵文字ピッカーを開き、リアクションを追加できることを確認する。

### 前提

TC-004-02 で投稿が1件以上存在する状態で実行する。

### 手順

```
Step 1: 掲示板ページに遷移する
  Tool: browser_navigate
  Params: { "url": "http://localhost:3900/trips/1/board" }
  期待結果: 掲示板ページが表示され、投稿が1件以上表示される

Step 2: スナップショットを取得して投稿と + ボタンを確認する
  Tool: browser_snapshot
  期待結果:
    - 投稿カードが表示されている
    - 各投稿のリアクションエリアに「+」ボタンが表示されている

Step 3: 投稿の + ボタンをクリックして絵文字ピッカーを開く
  Tool: browser_click
  Params: { "element": "投稿のリアクション追加ボタン（+）" }
  ※ スナップショットの ref 番号を使用する
  期待結果: 絵文字ピッカーが表示される

Step 4: 絵文字ピッカーの表示を確認する
  Tool: browser_snapshot
  期待結果:
    - 絵文字ピッカーが投稿の近くにポップアップ表示される
    - 6種類の絵文字オプションが表示される（表示されるもの: 笑顔, 親指, ハート, クラッカー, キラキラ, 笑い）

Step 5: 親指の絵文字をクリックする
  Tool: browser_click
  Params: { "element": "親指の絵文字ボタン" }
  ※ スナップショットの ref 番号を使用する
  期待結果: リアクションが追加される

Step 6: リアクション追加後の状態を確認する
  Tool: browser_snapshot
  期待結果:
    - 絵文字ピッカーが閉じている
    - 投稿のリアクションエリアにリアクションが表示される（親指アイコン + カウント「1」）
    - 自分が付けたリアクションはハイライト表示（border-primary-300 bg-primary-50）される
```

### 判定基準

- + ボタンクリックで絵文字ピッカーが開く
- 絵文字クリックでリアクションが投稿に追加される
- リアクションにカウント（1）が表示される
- 自分が付けたリアクションはハイライト表示される
- 絵文字ピッカーはリアクション追加後に自動で閉じる

## TC-004-05: 複数投稿の表示確認

### 目的

追加のメッセージを投稿し、複数件の投稿が正しく一覧表示されることを確認する。

### 前提

TC-004-02 で投稿が1件以上存在する状態で実行する。

### 手順

```
Step 1: 掲示板ページに遷移する
  Tool: browser_navigate
  Params: { "url": "http://localhost:3900/trips/1/board" }
  期待結果: 掲示板ページが表示され、既存の投稿が表示される

Step 2: スナップショットを取得して既存投稿を確認する
  Tool: browser_snapshot
  期待結果: 既存の投稿（「明日の集合場所はどこにする？」）が表示されている

Step 3: テキストエリアに新しいメッセージを入力する
  Tool: browser_click
  Params: { "element": "テキストエリア（placeholder='メッセージを入力...'）" }
  ※ スナップショットの ref 番号を使用する

  Tool: browser_type
  Params: { "text": "伊勢市駅の改札前でいいかな" }
  期待結果: テキストエリアに値が入力される

Step 4: 送信ボタンをクリックする
  Tool: browser_click
  Params: { "element": "送信ボタン（丸型 FAB）" }
  ※ スナップショットの ref 番号を使用する
  期待結果: 投稿処理が開始される

Step 5: 複数投稿の表示を確認する
  Tool: browser_snapshot
  期待結果:
    - 投稿一覧に2件の投稿が表示される
    - 「明日の集合場所はどこにする？」の投稿が表示されている
    - 「伊勢市駅の改札前でいいかな」の投稿が表示されている
    - 投稿は新しい順に並んでいる（「伊勢市駅の改札前でいいかな」が上に表示される）
    - テキストエリアがクリアされている
```

### 判定基準

- 2件の投稿が投稿一覧に表示される
- 新しい投稿が上に表示される（新しい順ソート）
- 各投稿にユーザー名・本文・タイムスタンプが表示される
- 投稿後にテキストエリアがクリアされる

## 実行時の注意事項

### テスト実行順序

テストケースは番号順に実行することを推奨する。特に以下の依存関係に注意する:

1. TC-004-01 は独立して実行可能（ただしログイン済みであること）
2. TC-004-02 は投稿を作成するため、TC-004-04 と TC-004-05 の前提になる
3. TC-004-03 は独立して実行可能
4. TC-004-04 は投稿が存在する状態が必要（TC-004-02 の後に実行）
5. TC-004-05 は TC-004-02 で作成した投稿が存在する状態が必要

### 推奨実行順

1. TC-004-01（ページ表示確認）
2. TC-004-03（空メッセージの送信防止）
3. TC-004-02（メッセージの投稿）
4. TC-004-04（リアクションの追加）
5. TC-004-05（複数投稿の表示確認）

### ログイン前提

全テストケースはログイン済みの状態で実行する。未認証状態でアクセスした場合は `/login` にリダイレクトされる（認証ミドルウェアが設定されているため）。テスト開始前に TC-001-02 の手順でたろうとしてログインしておくこと。

### browser_snapshot の ref 番号について

- `browser_snapshot` で取得したスナップショットには、各インタラクティブ要素に ref 番号が割り当てられる
- `browser_click` や `browser_type` では、この ref 番号を使って要素を指定する
- 本仕様書では要素の説明テキストで記載しているが、実行時はスナップショットの ref 番号に置き換えること

### エラー発生時の対応

- MCP ツールの呼び出しでエラーが返された場合は、即座にエラー内容を報告する
- 回避策の実行やリトライは行わない（CLAUDE.md の Playwright MCP 使用ルールに準拠）
- 期待結果と異なる場合は、実際の状態をスクリーンショットまたはスナップショットで記録し、差異を報告する
