# E2E テストシナリオ: 005 アルバム

## 概要

| 項目 | 内容 |
|------|------|
| シナリオID | 005 |
| 機能 | 写真アルバム（アップロード・表示・カバー設定・削除） |
| 対象ページ | `/trips/:tripId/album` |
| 関連API | `GET /api/trips/{tripId}/photos`, `POST /api/trips/{tripId}/photos`, `DELETE /api/trips/{tripId}/photos/{id}`, `PATCH /api/trips/{tripId}`, `GET /api/trips/{tripId}/spots` |
| 前提条件 | アプリが起動済み（フロントエンド: port 3900、バックエンド: port 8900）、ログイン済み、旅行が1件以上存在（tripId=1） |
| テストユーザー | たろう (`taro@example.com` / `password`) |

---

## 前提条件

- フロントエンド（Nuxt）が `http://localhost:3900` で起動していること
- バックエンド（Laravel）が `http://localhost:8900` で起動していること
- `php artisan migrate --seed` が実行済みで、テストユーザーが存在すること
- テストユーザー「たろう」でログイン済みであること（001_ログイン TC-001-02 参照）
- 旅行データが1件以上存在し、tripId=1 でアクセスできること

---

## テストケース一覧

| # | ケース名 | 種別 | 概要 |
|---|---------|------|------|
| TC-005-01 | アルバムページの表示 | 正常系 | アルバムページが正しく表示される |
| TC-005-02 | アップロードフォームの表示/非表示 | 正常系 | トグルボタンで PhotoUploader の表示/非表示が切り替わる |
| TC-005-03 | 写真のアップロード | 正常系 | 写真ファイルを選択してアップロードし、グリッドに追加される |
| TC-005-04 | 写真のライトボックス表示 | 正常系 | グリッド上の写真をクリックするとライトボックスで拡大表示される |
| TC-005-05 | カバー画像の設定 | 正常系 | ライトボックスから写真を旅行のカバー画像に設定できる |
| TC-005-06 | 写真の削除 | 正常系 | ライトボックスから写真を削除でき、グリッドから消える |

## TC-005-01: アルバムページの表示

### 目的

アルバムページが正しくレンダリングされ、ヘッダー、スライドショーリンク、アップロードボタン、スポットフィルターが全て表示されることを確認する。

### 手順

```
Step 1: アルバムページに遷移する
  Tool: browser_navigate
  Params: { "url": "http://localhost:3900/trips/1/album" }
  期待結果: ページが正常にロードされる

Step 2: ページのスナップショットを取得して UI 要素を確認する
  Tool: browser_snapshot
  期待結果: 以下の要素が存在する
    - タイトル「アルバム」（h1 要素）
    - 「スライドショー」リンクボタン（/trips/1/album/slideshow へのリンク）
    - 「アップロード」ボタン
    - スポットフィルター（select 要素、「すべてのスポット」がデフォルト選択）
    - PhotoGrid 領域（写真がある場合は3カラムグリッド、ない場合は「写真はまだありません」メッセージ）
    - PhotoUploader は表示されていない（showUploader が false の初期状態）
```

### 判定基準

- ページタイトルに「アルバム」が含まれる
- ヘッダーに「スライドショー」リンクと「アップロード」ボタンが存在する
- スポットフィルターの select 要素が表示され、「すべてのスポット」が選択されている
- default レイアウト（ヘッダー・ナビあり）で表示されている

## TC-005-02: アップロードフォームの表示/非表示

### 目的

「アップロード」ボタンのクリックで PhotoUploader コンポーネントの表示/非表示がトグルされ、ボタンテキストが「アップロード」と「閉じる」で切り替わることを確認する。

### 手順

```

  Tool: browser_navigate
  Params: { "url": "http://localhost:3900/trips/1/album" }
  期待結果: ページが正常にロードされる

Step 2: 初期状態を確認する
  Tool: browser_snapshot
  期待結果:
    - 「アップロード」ボタンが表示されている
    - PhotoUploader（ファイル選択エリア）は表示されていない

Step 3: 「アップロード」ボタンをクリックする
  Tool: browser_click
  Params: { "element": "アップロードボタン" }
  ※ スナップショットの ref 番号を使用する
  期待結果: PhotoUploader が表示される

Step 4: PhotoUploader の表示を確認する
  Tool: browser_snapshot
  期待結果:
    - ボタンテキストが「閉じる」に変わっている
    - ファイル選択エリア（「タップして写真を選択」テキスト）が表示されている
    - 点線のボーダーで囲まれたドロップエリアが表示されている

Step 5: 「閉じる」ボタンをクリックする

  Params: { "element": "閉じるボタン" }
  ※ スナップショットの ref 番号を使用する
  期待結果: PhotoUploader が非表示になる

Step 6: 非表示状態を確認する
  Tool: browser_snapshot

    - ファイル選択エリアが表示されていない
```

- ボタンクリックで PhotoUploader の表示/非表示がトグルされる
- ボタンテキストが状態に応じて「アップロード」/「閉じる」に切り替わる

## 実行時の注意事項

### テスト実行順序

テストケースは番号順に実行することを推奨する。特に以下の依存関係に注意する:

1. TC-005-01 は独立して実行可能（写真がなくてもページ表示は確認できる）
2. TC-005-02 は独立して実行可能
3. TC-005-03 は写真をアップロードするため、TC-005-04 ~ TC-005-06 の前提になる
4. TC-005-04 は少なくとも1枚の写真が必要（TC-005-03 の後に実行）
5. TC-005-05 は少なくとも1枚の写真が必要（TC-005-03 の後に実行）
6. TC-005-06 は写真を削除するため、最後に実行する

### 推奨実行順

1. TC-005-01（ページ表示確認）
2. TC-005-02（アップロードフォームのトグル）
3. TC-005-03（写真アップロード）
4. TC-005-04（ライトボックス表示）
5. TC-005-05（カバー画像設定）
6. TC-005-06（写真削除）

### テスト用画像ファイルについて

- テスト用の画像ファイルは `tests/fixtures/test_photo.jpg` に配置する
- JPEG 形式で、ファイルサイズは 10MB 以下とする
- `browser_file_upload` ツールでファイルパスを指定してアップロードする
- 実際の R2 ストレージへのアップロードが発生するため、テスト環境では R2 接続が必要

### confirm ダイアログについて

- TC-005-06 の削除操作ではブラウザの `confirm()` ダイアログが表示される
- `browser_handle_dialog` ツールでダイアログを承認（accept: true）する必要がある
- ダイアログが表示されるタイミングに注意すること

### browser_snapshot の ref 番号について

- `browser_snapshot` で取得したスナップショットには、各インタラクティブ要素に ref 番号が割り当てられる
- `browser_click` や `browser_type` では、この ref 番号を使って要素を指定する
- 本仕様書では要素の説明テキストで記載しているが、実行時はスナップショットの ref 番号に置き換えること

### エラー発生時の対応

- MCP ツールの呼び出しでエラーが返された場合は、即座にエラー内容を報告する
- 回避策の実行やリトライは行わない（CLAUDE.md の Playwright MCP 使用ルールに準拠）
- 期待結果と異なる場合は、実際の状態をスクリーンショットまたはスナップショットで記録し、差異を報告する
