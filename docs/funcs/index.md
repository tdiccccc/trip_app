# 機能仕様一覧

## 概要

伊勢旅行メモリーアプリの全機能一覧。
複数旅行管理に対応し、各機能は旅行に紐づいて管理される。
各機能の実装状況と対応するページ・API を記載する（最終更新: 2026-03-12）。

## 機能詳細

### 1. ログイン

- Sanctum SPA 認証（Cookie ベース）
- CSRF トークン取得 → メール + パスワードでログイン
- 認証済みユーザーは `/trips` へ自動リダイレクト
- ログアウト機能あり（`useAuth` composable 経由）

### 2. 旅行管理

- 旅行一覧表示（自分が参加している旅行をカード形式で表示）
- 旅行新規作成（タイトル、説明、行き先、開始日、終了日、メンバー追加）
- 旅行詳細表示（メンバー情報含む）
- 旅行情報の編集（タイトル、日程、行き先等）
- 旅行の削除（紐づく全データも削除）
- ログイン後のランディングページは旅行一覧（`/trips`）
- 権限制御: trip の編集・削除は作成者（owner）のみ可能。member には編集・削除ボタンを非表示

### 3. カウントダウン

- 旅行開始日までの日・時・分・秒をリアルタイム表示
- 1秒間隔で更新
- 旅行開始日を過ぎた場合は「Have a great trip!」メッセージを表示
- しおり・アルバムへのクイックリンク
- 旅行情報（タイトル、日程、行き先、メンバー）の表示

### 4. しおり（タイムライン）

- 旅行に紐づく行程を管理
- 日付タブで旅行日程に基づく日切り替え
- 行程を時系列で一覧表示（sort_order 順）
- モーダルフォームから行程の追加・編集
- 各行程の削除（確認ダイアログ付き）
- 当日は現在時刻に該当する行程を自動ハイライト（1分間隔更新）

### 5. スポット詳細

- 旅行に紐づくスポット情報を表示
- ヒーロー画像表示（画像がない場合はプレースホルダー）
- カテゴリ表示（観光・グルメ・宿泊・その他）
- 住所・営業時間・料金情報のカード表示
- Google Maps への外部リンク
- スポットに紐づく写真のグリッド表示
- メモの一覧表示・新規追加

### 6. 写真アルバム

- 旅行に紐づく写真を管理
- 写真のグリッド表示（PhotoGrid コンポーネント）
- 写真アップロード（PhotoUploader コンポーネント、spot_id / caption / taken_at 指定可能）
- スポット別フィルタリング（スポット一覧を API から取得）
- 写真タップで全画面 Lightbox 表示（インライン実装）
- Lightbox 内から写真削除

### 7. スライドショー

- 旅行のアルバム写真を全画面レイアウトで自動再生
- 撮影日時順（asc）で写真を取得
- トランジション切り替え（フェード / スライド）
- 再生速度の調整
- 再生・一時停止・前へ・次へのコントロール
- コントロール UI は3秒後に自動非表示、画面タップで再表示
- 写真がない場合のエンプティステート表示

### 8. ふたりの掲示板

- 旅行に紐づくメッセージボード
- テキストメッセージの投稿
- 投稿一覧表示（新しい順、自分の投稿は右寄せ）
- 絵文字リアクション機能（6種類のプリセット）
- リアクション数の集計表示（自分が付けたリアクションはハイライト）
- 投稿フォームは画面下部に固定配置

### 9. パッキングリスト

- 旅行に紐づく持ち物チェックリスト
- チェックボックス形式の持ち物リスト
- 担当者フィルタリング（全て / たろう / はなこ）
- 準備状況のプログレスバー表示
- アイテムの新規追加（名前 + 担当者選択）
- チェック状態のトグル（チェック済みは半透明 + 取り消し線）
- アイテムの削除（確認ダイアログ付き）

### 10. 費用メモ

- 旅行に紐づく支出を管理
- 支出の新規追加（項目名・金額・カテゴリ・支払者・メモ）
- カテゴリのカスタム管理（旅行ごとに作成・編集・削除が可能）
  - デフォルトカテゴリ: 食事・交通・お土産・宿泊・チケット・その他
  - カテゴリにはカラーコードを設定可能（UI での色分け表示に使用）
  - 費用記録が紐づいているカテゴリは削除不可（409 Conflict）
- 支払者: たろう / はなこ
- サマリーカード（合計金額・一人あたり・カテゴリ別内訳）
- カテゴリ別フィルタリング
- 支出の削除（確認ダイアログ付き）

### 11. エクスポート

- 旅行の記録を各種形式で出力
- PDF しおり・フォトブック PDF・ZIP 一括ダウンロードの3機能は Blob ダウンロードで実装済み
- スライドショー動画（MP4）は「準備中」表示（バックエンド API が 501 スタブ）
- 各カードにアイコン・説明・ダウンロードボタン・ローディング状態を実装
- エラーハンドリングあり（ダウンロード失敗時のエラー表示）

### 12. 権限制御強化（ロック機能）

- 旅行メタ情報の編集・削除は owner のみ（実装済み）
- 写真の削除は投稿者本人のみに制限（要実装）
- 掲示板投稿の削除は投稿者本人のみに制限（要実装）
- しおり・スポット・パッキング・費用は全メンバーが共同編集可能
- 閲覧は全メンバーに許可
- APIレベルで403 Forbiddenを返却（UIの非表示だけでなくdefense in depth）
- 優先度: P1

### 13. 観光ルート記録

- しおりページに「リスト / マップ」表示切替タブを追加
- Leaflet + OpenStreetMap による地図表示（無料、APIキー不要）
- itinerary_items に紐づくスポットを番号付きマーカーで表示
- スポット間を polyline で接続（ルート線）
- 日付タブとの連動（日別のスポットのみ表示）
- マーカータップでスポット名・時刻のポップアップ表示
- 全スポットが収まる表示範囲の自動調整（fitBounds）
- 新規API・テーブル不要（既存の spots lat/lng + itinerary_items の順序を利用）
- 優先度: P2

### 14. 今どこ

- navigator.geolocation.watchPosition で現在地をリアルタイム取得
- 全スポットとの Haversine 距離を計算し、最寄りスポットを表示
- 距離に応じた表現（200m以内: 「にいます」、200m〜2km: 「の近く」、2km超: 距離表示）
- しおりの行程順から「次のスポット」を特定し、距離を表示
- 当日の行程ミニタイムライン（済/次/未のステータス付き）
- 次のスポットへの Google Maps リンク（ナビ起動用）
- GPS取得失敗・権限拒否時のフォールバック表示
- 画面非アクティブ時はGPS追跡を一時停止（バッテリー対策）
- フロントエンドのみで完結（新規API・テーブル不要）
- 優先度: P2

### 15. 旅行サマリー

- 旅行の各種統計情報をカードグリッドで一覧表示
- 表示項目（MVP）: 写真枚数、訪問スポット数、掲示板投稿数、リアクション合計、費用合計、ひとりあたり費用、最初/最後の写真タイムスタンプ
- 表示項目（Nice-to-have）: パッキング達成率、行程数、日別写真枚数、費用カテゴリ別内訳
- 専用 API（GET /api/trips/:tripId/summary）で全統計を1リクエストで取得
- 新規テーブル不要（既存テーブルのCOUNT/SUM/MIN/MAXで集計）
- 旅行トップページのクイックリンクに「サマリー」を追加
- 優先度: P2
